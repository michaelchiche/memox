# memo

A CLI tool for organizing markdown notes with LLM assistance.

## Features

- **Topic-based organization**: Group notes into topics instead of flat tags
- **LLM-assisted classification**: Automatically suggests topics for your notes using local LLM (Ollama)
- **LLM generation tracking**: Track which model was used for each classification and summary
- **Bidirectional sync**: Database and markdown files stay in sync
- **Dynamic summaries**: Generate AI summaries for topics that update over time
- **Versioned summaries**: Track how topics evolve over time
- **Full-text search**: Find notes quickly with SQLite FTS5
- **Hierarchical config**: Project-local and global configuration files

## Installation

```bash
npm install -g memo
```

## Prerequisites

- Node.js 18 or higher
- [Ollama](https://ollama.ai) (for LLM features)

## Quick Start

```bash
# Initialize the database
memo sync

# Import notes from a directory
memo import ./my-notes/

# List all topics
memo topic list

# View a topic and its notes
memo topic show "Project Alpha"

# Create a new topic
memo topic create "My Project"

# Generate a summary for a topic
memo topic summarize "My Project"
```

## Commands

### Note Commands

```bash
# Import notes (interactive, with LLM suggestions)
memo import ./path/to/notes/

# Import without prompts (assign to default topic)
memo import ./notes/ --auto

# Show a note's details
memo note show meeting-notes.md

# Show a note with LLM model used for classification
memo note show meeting-notes.md --show-llm

# List topics for a note
memo note topics project-plan.md
```

### Topic Commands

```bash
# List all topics
memo topic list

# Show topic details and notes
memo topic show "Project Alpha"

# Show topic with LLM model used for summary generation
memo topic show "Project Alpha" --show-llm

# Create a new topic
memo topic create "New Topic"

# Create with description
memo topic create "New Topic" --description "My awesome topic"

# Rename a topic
memo topic rename "Old Name" "New Name"

# Delete a topic
memo topic delete "Topic Name"

# Generate/regenerate summary
memo topic summarize "Project Alpha"

# View summary history
memo topic history "Project Alpha"

# LLM proposes topics from unorganized notes
memo topic propose
```

### Sync Commands

```bash
# Synchronize database with files
memo sync
```

### Config Commands

```bash
# View current configuration
memo config

# Show active config file path
memo config --path

# Set a config value (local config)
memo config set llm.model mistral

# Set a config value (global config)
memo config set --global llm.model mistral

# View a specific config key
memo config llm.model
```

## Configuration

Memo searches for configuration files in this order (highest priority first):

1. **Environment variables**
   - `MEMO_CONFIG_PATH` - Use a specific config file
   - `MEMO_STORE_PATH` - Override store path
   - `MEMO_DB_PATH` - Override database path
   - `MEMO_LLM_MODEL` - Override LLM model

2. **Local config files** (walked up from current directory to home)
   - `.memorc.yaml` (hidden, takes priority)
   - `memo.yaml` (visible, secondary)

3. **Global fallback**
   - `~/.memorc.yaml`

### Example config files

```yaml
# ~/.memorc.yaml (global config)
storePath: ~/memo/store
dbPath: ~/memo/memo.db
llm:
  provider: ollama
  model: llama3.2
  baseUrl: http://localhost:11434
defaultTopic: Non classée
```

```yaml
# ./memo.yaml (project-local config)
llm:
  model: mistral  # Override model for this project
```

### Project-local Configuration

You can have different configurations per project by creating `memo.yaml` or `.memorc.yaml` in your project directory:

```
~/projects/
├── project-a/
│   ├── memo.yaml        # Uses model: llama3.2
│   └── notes/
├── project-b/
│   ├── .memorc.yaml     # Uses model: mistral
│   └── notes/
└── my-notes/
    └── (no config)      # Uses ~/.memorc.yaml
```

### Debug Mode

To see which config file is being used:

```bash
MEMO_DEBUG=1 memo topic list
# [DEBUG] Using config: /home/user/projects/myapp/.memorc.yaml
```

## How It Works

### Notes and Topics

Notes are markdown files with optional YAML frontmatter:

```markdown
---
title: Meeting Notes
topics:
  - Project Alpha
  - Meetings
created: 2024-03-15
---

# Meeting Notes

Content goes here...
```

The database maintains relationships between notes and topics. When you add a note to a topic, both the database and the file are updated.

### LLM Integration

When importing notes, the LLM analyzes content and suggests:

1. **Existing topics** (ranked by confidence)
2. **New topics** (proposed based on content)

You can accept suggestions, enter custom topics, or skip.

### Summaries

Topic summaries are generated by the LLM and versioned over time. Each generation adds a new version, allowing you to track how topics evolve.

### LLM Generation Tracking

Memo tracks which LLM model was used for each generation:

- **Note classification**: The model used to classify a note is stored in both the database and the note's frontmatter
- **Topic summaries**: The model used to generate a summary is stored in the database

Frontmatter example with LLM metadata:

```markdown
---
title: Meeting Notes
topics:
  - Project Alpha
created: 2024-03-15
generated_by:
  provider: ollama
  model: llama3.2
---

# Meeting Notes

Content goes here...
```

Use `--show-llm` flag to view the model used:

```bash
memo note show meeting-notes.md --show-llm
memo topic show "Project Alpha" --show-llm
```
